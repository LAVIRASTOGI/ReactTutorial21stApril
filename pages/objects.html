Objects in JavaScript

Objects are one of the most important data types in JavaScript. They allow you to store collections of data and more complex entities.

In JavaScript, an object is a collection of key-value pairs, where keys are strings (or Symbols) and values can be any data type, including other objects.

Creating Objects

There are multiple ways to create objects in JavaScript:

// Object literal (most common)
const person = {
  name: 'John',
  age: 30,
  isEmployed: true
};

// Using the Object constructor
const car = new Object();
car.make = 'Toyota';
car.model = 'Camry';
car.year = 2020;

// Using Object.create()
const animal = Object.create(null); // Creates an object with no prototype
animal.species = 'Dog';
animal.sound = 'Woof';

Accessing Object Properties

You can access object properties in two ways:

// Dot notation
console.log(person.name); // "John"
console.log(car.make);    // "Toyota"

// Bracket notation (useful when property names are dynamic or contain special characters)
console.log(person['age']);     // 30
const propertyName = 'model';
console.log(car[propertyName]); // "Camry"

// Accessing non-existent properties returns undefined
console.log(person.address);    // undefined

Modifying Objects

Objects in JavaScript are mutable and can be modified:

// Adding new properties
person.address = '123 Main St';
person['email'] = 'john@example.com';

// Modifying existing properties
person.age = 31;

// Deleting properties
delete person.isEmployed;

console.log(person);
// { name: 'John', age: 31, address: '123 Main St', email: 'john@example.com' }

Methods in Objects

Objects can contain functions as properties, which are called methods: 

const person = {
  name: 'Alice',
  age: 25,
  // Method defined using function keyword
  greet: function() {
    return `Hello, my name is ${this.name}!`;
  },
  // Method defined using shorthand syntax (ES6)
  introduce() {
    return `I am ${this.age} years old.`;
  }
};

console.log(person.greet());     // "Hello, my name is Alice!"
console.log(person.introduce()); // "I am 25 years old."

Object Properties and Methods

JavaScript provides several built-in methods for working with objects: 

const user = {
  id: 1,
  username: 'alice123',
  email: 'alice@example.com',
  isAdmin: false
};

// Object.keys() - returns an array of the object's own enumerable property names
console.log(Object.keys(user));
// ["id", "username", "email", "isAdmin"]

// Object.values() - returns an array of the object's own enumerable property values
console.log(Object.values(user));
// [1, "alice123", "alice@example.com", false]

// Object.entries() - returns an array of [key, value] pairs
console.log(Object.entries(user));
// [["id", 1], ["username", "alice123"], ["email", "alice@example.com"], ["isAdmin", false]]

// Object.assign() - copies properties from one object to another
const defaults = { theme: 'light', notifications: true };
const preferences = { theme: 'dark' };
const mergedSettings = Object.assign({}, defaults, preferences);
console.log(mergedSettings);
// { theme: "dark", notifications: true }

Object.freeze() and Object.seal()

Methods to control object mutability:

// Object.freeze() - prevents adding, removing, or changing properties
const frozenObj = { x: 1, y: 2 };
Object.freeze(frozenObj);

// These operations will fail silently (or throw error in strict mode)
frozenObj.x = 10;      // No change
frozenObj.z = 3;       // No addition
delete frozenObj.y;    // No deletion
console.log(frozenObj); // { x: 1, y: 2 }

// Object.seal() - prevents adding or removing properties, but allows changing existing ones
const sealedObj = { a: 1, b: 2 };
Object.seal(sealedObj);

sealedObj.a = 10;      // Allowed (changing existing property)
sealedObj.c = 3;       // No addition
delete sealedObj.b;    // No deletion
console.log(sealedObj); // { a: 10, b: 2 }

Computed Property Names (ES6)

You can use expressions as property names in object literals:

// Computed property names
const propertyName = 'age';
const propertyPrefix = 'user';

const person = {
  name: 'Bob',
  [propertyName]: 35,              // Use variable as property name
  [`${propertyPrefix}Id`]: 'U123'  // Use expression as property name
};

console.log(person.age);     // 35
console.log(person.userId);  // "U123"

Property Shorthand (ES6)

When variables have the same name as object properties:

// Property shorthand
const name = 'Charlie';
const age = 40;
const job = 'Developer';

// Instead of { name: name, age: age, job: job }
const user = { name, age, job };

console.log(user); // { name: "Charlie", age: 40, job: "Developer" }

Object Destructuring (ES6)

Extracting object properties into variables:

const product = {
  id: 'P123',
  name: 'Laptop',
  price: 999,
  specs: {
    ram: '16GB',
    storage: '512GB SSD',
    cpu: 'i7'
  }
};

// Basic destructuring
const { name, price } = product;
console.log(name, price); // "Laptop" 999

// Destructuring with new variable names
const { id: productId, specs: specifications } = product;
console.log(productId, specifications); // "P123" { ram: "16GB", storage: "512GB SSD", cpu: "i7" }

// Nested destructuring
const { specs: { ram, cpu } } = product;
console.log(ram, cpu); // "16GB" "i7"

// Default values
const { category = 'Electronics' } = product;
console.log(category); // "Electronics" (not in object, so default value is used)
