JavaScript Variables: let, const, var

JavaScript provides three ways to declare variables: var, let, and const. Each has different scoping rules and behaviors that affect how they can be used in your code.

var

var is the oldest way to declare variables in JavaScript. It has function scope (or global scope if declared outside a function). 

// Function scope example
function varExample() {
  var x = 10;
  if (true) {
    var x = 20;  // Same variable!
    console.log(x); // 20
  }
  console.log(x); // 20 - the value was changed
}

// Hoisting example
console.log(y); // undefined (not an error)
var y = 5;      // Declaration is hoisted, but not the initialization

Key characteristics of var:

  • Function scoped (or global if outside a function)
  • Can be redeclared without error
  • Hoisted to the top of its scope (declaration only, not initialization)
  • Can be updated

let

let was introduced in ES6 (2015) and provides block scope for variables. 

// Block scope example
function letExample() {
  let x = 10;
  if (true) {
    let x = 20;  // Different variable (block scoped)
    console.log(x); // 20
  }
  console.log(x); // 10 - the original value is unchanged
}

// Temporal Dead Zone example
// console.log(z); // ReferenceError: Cannot access 'z' before initialization
let z = 5;

// Can be updated
let count = 1;
count = 2; // Valid

Key characteristics of let:

  • Block scoped (available only within the block it's defined)
  • Cannot be redeclared in the same scope
  • Hoisted but not initialized (Temporal Dead Zone)
  • Can be updated

const

const also introduced in ES6, creates block-scoped constants that cannot be reassigned. 

// Basic const usage
const PI = 3.14159;
// PI = 3.14; // TypeError: Assignment to constant variable

// Object mutability
const person = { name: "Alice" };
person.name = "Bob"; // Valid - the object can be mutated
// person = { name: "Charlie" }; // TypeError - reassignment is not allowed

// Array mutability
const numbers = [1, 2, 3];
numbers.push(4); // Valid - we can modify the array
// numbers = [5, 6, 7]; // TypeError - reassignment is not allowed

Key characteristics of const:

  • Block scoped (like let)
  • Cannot be redeclared in the same scope
  • Must be initialized when declared
  • Cannot be reassigned
  • For objects and arrays, the contents can be modified (mutable)

Scope Comparison

function scopeExample() {
  // Global scope
  var globalVar = "I'm global (within function)";
  let globalLet = "I'm also global (within function)";
  
  // Block scope
  {
    var blockVar = "I'm still accessible outside the block";
    let blockLet = "I'm only accessible within this block";
    const blockConst = "I'm also only accessible within this block";
    
    console.log(globalVar);    // Accessible
    console.log(globalLet);    // Accessible
    console.log(blockVar);     // Accessible
    console.log(blockLet);     // Accessible
    console.log(blockConst);   // Accessible
  }
  
  console.log(globalVar);    // Accessible
  console.log(globalLet);    // Accessible
  console.log(blockVar);     // Accessible
  // console.log(blockLet);   // ReferenceError
  // console.log(blockConst); // ReferenceError
}

Best Practices

  • Avoid using var in modern JavaScript code
  • Use const by default (for values that won't be reassigned)
  • Use let when you need to reassign values
  • Declare variables at the top of their scope for clarity
  • Use descriptive variable names
