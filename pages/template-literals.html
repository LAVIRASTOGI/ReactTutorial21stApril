Template Literals in JavaScript

Template literals, introduced in ES6 (ES2015), provide an improved way to work with strings in JavaScript. They offer string interpolation, multi-line strings, and tagged templates without the need for concatenation or escape characters.

Template literals are enclosed by backtick (`) characters instead of single or double quotes, making them more versatile than traditional string literals.

Basic Syntax

Template literals are created using backticks (`) instead of quotes: 

// Traditional string with quotes
const message1 = "Hello, world!";
const message2 = 'Hello, world!';

// Template literal with backticks
const message3 = `Hello, world!`;

console.log(message1); // "Hello, world!"
console.log(message2); // "Hello, world!"
console.log(message3); // "Hello, world!"

String Interpolation

Template literals allow embedding expressions using ${expression} syntax: 

// Traditional string concatenation
const name = "Alice";
const age = 28;
const greeting1 = "Hello, my name is " + name + " and I am " + age + " years old.";

// Using template literals
const greeting2 = `Hello, my name is ${name} and I am ${age} years old.`;

console.log(greeting1); // "Hello, my name is Alice and I am 28 years old."
console.log(greeting2); // "Hello, my name is Alice and I am 28 years old."

// Expressions in template literals
console.log(`2 + 2 = ${2 + 2}`); // "2 + 2 = 4"
console.log(`Today is ${new Date().toLocaleDateString()}`);

// Using functions in expressions
const capitalize = str => str.charAt(0).toUpperCase() + str.slice(1);
console.log(`Hello, ${capitalize(name)}`); // "Hello, Alice"

Multi-line Strings

Template literals preserve line breaks, making multi-line strings easy to write: 

// Traditional multi-line strings (with escape characters)
const multiLine1 = "Line 1\nLine 2\nLine 3";

// Multi-line template literal
const multiLine2 = `Line 1
Line 2
Line 3`;

console.log(multiLine1);
// Line 1
// Line 2
// Line 3

console.log(multiLine2);
// Line 1
// Line 2
// Line 3

// Multi-line with interpolation
const title = "Features";
const list = `ES6 ${title}:
1. Template literals
2. Arrow functions
3. Destructuring
4. Classes`;

console.log(list);
// ES6 Features:
// 1. Template literals
// 2. Arrow functions
// 3. Destructuring
// 4. Classes

const userCard = `

Tagged Templates

Tagged templates allow you to parse template literals with a function: 

// Tagged template function
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    const value = values[i] || '';
    const highlighted = typeof value === 'number' 
      ? `${value}` 
      : value;
    return result + str + highlighted;
  }, '');
}

// Using the tag function
const price = 19.99;
const product = 'JavaScript Book';
const message = highlight`The ${product} costs $${price}!`;

console.log(message);
// "The JavaScript Book costs $19.99!"

// More complex tagged template example
function sql(strings, ...values) {
  // Create a prepared statement with placeholders
  const statement = strings.reduce((result, str, i) => {
    return result + str + (i < values.length ? '?' : '');
  }, '');
  
  // Return an object with the statement and values
  return {
    statement,
    values,
    execute() {
      return `Executing SQL: ${this.statement} with values: ${this.values}`;
    }
  };
}

const userId = 42;
const role = 'admin';
const query = sql`SELECT * FROM users WHERE id = ${userId} AND role = ${role}`;

console.log(query.statement); // "SELECT * FROM users WHERE id = ? AND role = ?"
console.log(query.values);    // [42, "admin"]
console.log(query.execute());  // "Executing SQL: SELECT * FROM users WHERE id = ? AND role = ? with values: 42,admin"

Escape Sequences

Template literals support escape sequences like traditional strings: 

// Using escape sequences in template literals
const path = `C:\\Documents\\Files`;  // Escaping backslashes
console.log(path); // "C:\Documents\Files"

// Escaping backticks
const code = `The backtick character (\`) is used for template literals`;
console.log(code); // "The backtick character (`) is used for template literals"

// Escaping ${} syntax
const syntax = `To write \${variable} literally, escape the $ symbol`;
console.log(syntax); // "To write ${variable} literally, escape the $ symbol"

Best Practices and Tips

Guidelines for using template literals effectively:

// 1. Use template literals for string interpolation
// Good
const message = `Hello, ${userName}!`;

// Avoid
const message = "Hello, " + userName + "!";

// 2. Use template literals for multi-line strings
// Good
const html = `
  

    
Title

    
Content

  

`;

// Avoid
const html = "
\n" +
  "  
Title
\n" +
  "  
Content
\n" +
  "
";

// 3. For complex HTML generation, consider using functions
function createUserCard(user) {
  return `
    

      
${user.name}

      
${user.bio}

    

  `;
}

// 4. Be careful with spaces and indentation
// This includes leading and trailing spaces
const indented = `
  This will have
  leading and trailing
  whitespace
`; 

// You can use trim() to remove unwanted whitespace
const trimmed = `
  This will be trimmed
  but maintain internal spacing
`.trim();

Raw Strings

Using String.raw to get the raw template string:

// String.raw for raw template literals
const filePath = String.raw`C:\Users\Documents\file.txt`;
console.log(filePath); // "C:\Users\Documents\file.txt"

// Without String.raw, the \U would be interpreted as an escape sequence
const windowsPath = `C:\Users\Documents\file.txt`;
console.log(windowsPath); // Would interpret \U, \D, etc. as escape sequences

// Raw tagged templates
function tag(strings, ...values) {
  console.log(strings.raw); // Array of raw strings
  return strings.raw[0];
}

const result = tag`Hello\nWorld`;
console.log(result); // "Hello\nWorld" (not interpreted as a newline)
